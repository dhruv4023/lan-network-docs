⚠ Before you start

Many thermal printers ship with the same default static IP. If you connect several at once before configuring them, you will get IP conflicts. Always configure printers one at a time, disconnected from each other, before joining them to the shared switch or router.

Keep one always‑on device (this laptop, or any PC) connected to the same wired network as the printers — some setups drop an idle printer from the network until traffic resumes.

DHCP reservation vs. static IP

DHCP reservation

On the printer's self‑test page, check if DHCP is enabled. If it is, reserve a fixed IP for that printer's MAC address inside the router's settings — no printer‑side changes needed.

Static IP on the printer

If DHCP is disabled or unsupported, open the printer's network settings page directly and assign the IP, subnet, gateway and DNS by hand.

Configure printers one at a time

  1. Connect only one printer directly to the laptop via Ethernet (no router or switch yet).
  2. Print a self‑test page and note the IP and whether DHCP is enabled.
  3. DHCP enabled → reserve its IP in the router and move to the next printer. DHCP disabled → continue below.
  4. Open a browser to the printer's IP and go to its Network Settings page.
    If the page won't open, set a manual IP on the laptop first:
    IP192.168.192.169 Subnet255.255.255.0 Gateway192.168.192.1 DNS8.8.8.8
  5. Assign a unique static IP (e.g. 192.168.1.11), subnet 255.255.255.0, gateway 192.168.1.1, DNS 8.8.8.8. Save, restart if required, then disconnect and repeat for the next printer.

Connect the hardware

  1. Connect the laptop to the router.
  2. Connect all configured printers to the same router.
  3. Power on the router and all printers.
  4. Not enough ports on the router? Add a network switch between the router and the printers.
Keep the link alive. Leave an always‑on wired device on the same network as the printer. Some setups make idle printers unreachable after a short period without an active device present.

Printer Configuration

Per‑printer walkthroughs: self‑test, static IP, DHCP reservation, and adding to ePOS Proxy.

1. Print a self‑test page

  1. Turn the printer off.
  2. Press and hold FEED.
  3. Turn the printer on while still holding FEED.
  4. Release once the page prints. Note the IP, subnet and gateway shown.

2. Static IP assignment walkthrough

  1. Connect only this printer to the laptop, isolated from other printers.
  2. Browse to the printer's current IP and open its Network Settings page.
  3. Disable DHCP if it's enabled.
  4. Enter a unique static IP, matching subnet mask, gateway and DNS for your topology.
  5. Save, restart the printer if prompted, then disconnect before configuring the next unit.

3. DHCP reservation walkthrough (router topology only)

  1. Check the self‑test page — if DHCP is enabled, leave it on.
  2. Log into the router's admin page and find DHCP reservations / static leases.
  3. Add a reservation pairing the printer's MAC address to a chosen IP inside the router's range.
  4. Reboot the printer; confirm it now always receives that reserved IP.

4. Add the printer to ePOS Proxy

  1. Open the ePOS Proxy application.
  2. Click + Add Network Printer.
  3. Enter the printer's IP address.
  4. Click Add. Repeat for every printer.

Troubleshooting

Expand a symptom to see likely causes and fixes.

Printer configuration page won't open in the browser

Your laptop's Ethernet adapter is probably on a different subnet than the printer.

  1. Open Network & Internet Settings → Ethernet → adapter properties → IPv4.
  2. Set IP 192.168.192.169, subnet 255.255.255.0, gateway 192.168.192.1, DNS 8.8.8.8.
  3. Save and retry the printer's IP in the browser.
Two printers stopped responding after connecting them to the switch

This is almost always an IP conflict — printers often ship with the same default address. Disconnect all printers, then reconnect and configure them one at a time with unique static IPs before joining the switch again.

A printer becomes unreachable after sitting idle

Some networks drop idle printers. Keep at least one always‑on wired device (the laptop or any PC) connected to the same network segment as the printer to keep the link active.

ePOS Proxy can't find a printer I just added
  1. Re‑print a self‑test page and confirm the printer's current IP hasn't changed.
  2. Ping the printer's IP from the laptop to confirm reachability.
  3. Re‑enter the exact IP in + Add Network Printer and click Add again.
Not enough Ethernet ports on the router for all printers

Add a network switch between the router and the printers; connect the switch to one router port, then connect all printers to the switch.

I don't know if my printer supports DHCP

Print a self‑test page — it will show whether DHCP is enabled. If you can't tell, treat it as disabled and assign a static IP manually via the printer's Network Settings page.

Frequently Asked Questions

Do I need a router for a single printer?

No. A single printer can be wired directly to your laptop with an Ethernet cable — no router or switch required.

Why do all my printers need the same subnet mask?

Devices can only talk directly to each other if they're on the same subnet. Mismatched subnet masks make printers unreachable even with correct IPs.

Can I reuse the same static IP plan across stores?

Yes — the IP assignment examples (192.168.1.10–.13, etc.) are a template. Just make sure no other device on that location's network already uses those addresses.

What's the difference between the switch‑only and router setups?

A switch‑only network has no internet gateway or DHCP server — every device needs a manually assigned static IP. A router setup can offer DHCP reservations, so the router itself remembers each printer's address.

What DNS server should I use?

The runbook uses Google's public DNS, 8.8.8.8, as the default preferred DNS for laptops and printers alike.
